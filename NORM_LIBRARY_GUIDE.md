# Руководство по наполнению Нормативной Базы (Norm Library)

Система "Digital Audit" позволяет оцифровывать нормативные документы (ГОСТ, СП, СН) и автоматически превращать их в интерактивные чек-листы для аудита.

## Шаг 1: Добавление документа

1. Перейдите в раздел **Библиотека норм** (Norm Library) через главное меню.
2. Нажмите кнопку **"Добавить норматив"** (с иконкой плюса) в верхнем правом углу.
3. В открывшемся окне заполните метаданные:
    *   **Юрисдикция**: KZ (Казахстан), RU (Россия) или INT (Международный).
    *   **Тип документа**: СН, СП, ГОСТ, Техрегламент.
    *   **Шифр / Номер**: Пример: `СН РК 2.02-01-2019`.
    *   **Название**: Полное название документа.
    *   **Дата редакции**: Дата введения или последнего изменения (необязательно).
4. **Загрузка файла**:
    *   Нажмите на поле загрузки и выберите **PDF** файл с текстом норматива.
    *   *Важно*: Файл должен содержать распознаваемый текстовый слой (не просто скан-картинка), чтобы система могла его прочитать.
5. Нажмите **"Создать"**. Документ появится в общем списке.

## Шаг 2: Парсинг (Оцифровка требований)

После создания документ еще "пустой" — система знает о файле, но не знает конкретных требований (пунктов).

1. Найдите документ в списке и нажмите **"Открыть"**.
2. Вы попадете на страницу детализации. Справа вы увидите раздел **"Требования и Разделы"**.
3. Нажмите кнопку **"Парсинг (Raw)"** (серая кнопка).
    *   Система прочитает PDF файл.
    *   Автоматически найдет пункты (например, `5.1.`, `6.2.3.`).
    *   Разобьет текст на отдельные требования.
4. После завершения (появится сообщение) обновите страницу или дождитесь появления списка требований.

## Шаг 3: AI Обогащение (Классификация)

Чтобы чек-лист был "умным" (понимал критичность и методы контроля), запустите AI-анализ.

1. На странице документа, рядом с кнопкой парсинга, нажмите **"AI Обогащение"** (цветная кнопка).
2. Система проанализирует текст каждого требования:
    *   **Severity (Критичность)**: Если в тексте есть слова "ЗАПРЕЩАЕТСЯ", "НЕ ДОПУСКАЕТСЯ", требование получит статус **Critical** (Красный).
    *   **Method (Метод)**: Если есть слова "ИЗМЕРИТЬ", "СОПРОТИВЛЕНИЕ", метод станет **Measurement** (Линейка). Если "АКТ", "ПАСПОРТ" — **Document** (Документы).
    *   **Tags (Теги)**: Автоматически проставятся теги `#Cable`, `#Grounding`, `#Power` и т.д.

## Результат

Теперь этот норматив готов к использованию в проектах!

1. Перейдите в **Проекты** -> Выберите проект.
2. Нажмите **"Добавить раздел аудита"**.
3. В списке "Каталог требований" вы увидите ваш новый норматив (например, `RS-SN-RK-2.02...`).
4. При создании чек-листа все пункты из норматива автоматически попадут в план проверки аудитора.
