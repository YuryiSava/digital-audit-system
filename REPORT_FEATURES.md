# Руководство по функциям отчетов (Digital Audit System v2.0)

В систему добавлены новые возможности для генерации профессиональных отчетов с использованием ИИ.

## 1. Настройка отчета
На странице отчета нажмите кнопку **"Настройки"** (шестеренка).
В открывшемся окне можно:
- **Загрузить Логотип**: Рекомендуется PNG на прозрачном фоне.
- **Ввести данные**: Номер договора, ФИО аудитора, должность.
- **Описать объект**: Технические характеристики для раздела DOC-02.

## 2. AI Генерация выводов (✨ New!)
В окне настроек перейдите к разделу "Содержание отчета" и нажмите кнопку **"Автозаполнение AI"**.

Система использует **GPT-4o-mini**, чтобы проанализировать все выявленные нарушения и автоматически написать:
1.  **Заключение аудитора (Summary)**: Общий вердикт о состоянии системы.
2.  **Критические риски**: Главные угрозы безопасности.
3.  **План действий (Рекомендации)**: Первоочередные шаги по устранению.

> **Важно:** Вы можете (и должны) редактировать сгенерированный текст перед сохранением.

## 3. PDF Парсинг с AI
При загрузке нового норматива в Библиотеку Норм:
1.  Загрузите PDF.
2.  Система автоматически распознает структуру документа (Статьи, Пункты, Подпункты) с помощью **OpenAI GPT-4**.
3.  Таблицы и сложные списки преобразуются в чистый Markdown/Text.

## 4. Конфигурация API
Для работы функций AI необходимы ключи в файле `.env.local`:
- `OPENAI_API_KEY`: Для генерации отчетов и парсинга PDF (обязательно).
- `GEMINI_API_KEY`: НЕ используется в текущей версии системы.
